## Проекты по Classic ML и Computer Vision

Ниже представлено краткое описание каждого проекта и выполненных в его рамках задач.

<summary><strong>HW 1: Библиотека sklearn и классификация с помощью KNN — «Игра Престолов»</strong></summary>

*   **Тема:** Знакомство с полным ML-пайплайном на задаче бинарной классификации: предсказание выживаемости персонажей из вселенной «Игры Престолов».
*   **Задания:**
    *   **Часть 1. Анализ и предобработка данных:**
        *   Обработка пропущенных значений: визуализация через барчарты и тепловые карты, удаление неинформативных столбцов, заполнение медианой/модой.
        *   Создание новых признаков: `isPopular` (бинаризация popularity score), `boolDeadRelations` (наличие мёртвых родственников), группировка похожих значений категории `culture`.
        *   Кодирование категориальных признаков с помощью `OneHotEncoder`.
        *   Анализ целевой переменной, корреляций и влияния признаков (violin plots).
        *   Разбиение на train/test с применением тех же преобразований к тестовой выборке.
    *   **Часть 2. Обучение моделей:**
        *   Обучение 8 классификаторов из `sklearn`: `LogisticRegression`, `DecisionTreeClassifier`, `AdaBoostClassifier`, `RandomForestClassifier`, `GaussianProcessClassifier`, `GaussianNB`, `KNeighborsClassifier`, `SVC`.
        *   Подбор гиперпараметров для дерева решений и AdaBoost через `GridSearchCV`.
    *   **Часть 3. Оценка качества:**
        *   Сравнение всех моделей по метрике `accuracy`. Лучший результат показал `GaussianProcessClassifier` (~0.7789 на публичном тесте).
        *   Формирование файла `submission.csv` для сдачи.

<summary><strong>HW 2: Линейные алгоритмы — реализация логистической регрессии с нуля</strong></summary>

*   **Тема:** Глубокое понимание градиентного спуска и линейных моделей путём реализации с нуля на `NumPy`.
*   **Задания:**
    *   **Градиентный спуск:** реализация функции градиентного спуска для $f(x,y) = \sin^2x + \sin^2y$, аналитический расчёт частных производных; визуализация траектории спуска на интерактивном 3D-графике (`plotly`) и графике loss-per-step.
    *   **Генератор батчей:** реализация генератора батчей для SGD с перемешиванием датасета по эпохам; последний неполный батч отбрасывается.
    *   **Логистическая регрессия:** реализация класса `MyLogisticRegression` с методами `fit` и `get_grad`; используется SGD с генератором батчей; функция потерь — бинарная кросс-энтропия; градиент $\frac{\partial L}{\partial w} = X^T(p - y)$.
    *   **ElasticNet-регуляризация:** расширение класса через наследование; реализация `MyElasticLogisticRegression` с L1- и L2-регуляризацией, переопределение только метода `get_grad`.
    *   **Тестирование:** проверка на синтетических данных и на датасете MNIST (бинарная классификация цифр 0 и 1), оценка качества через кросс-валидацию.

</details>

<summary><strong>HW 3: Решение конкурса на Kaggle — полный ML-пайплайн</strong></summary>

*   **Тема:** Самостоятельное решение задачи бинарной классификации в соревновательном формате с оценкой по метрике ROC-AUC.
*   **Задания:**
    *   **Загрузка и первичный анализ данных:** просмотр случайных строк, поиск и обработка пропущенных значений (`isna()`, `info()`), визуализация доли пропусков по колонкам.
    *   **Анализ данных (EDA):** построение гистограмм и boxplot для числовых признаков; bar-диаграмм и pie-диаграмм для категориальных; анализ сбалансированности целевой переменной.
    *   **Линейные модели:** нормализация числовых признаков, OneHotEncoding категориальных; подбор гиперпараметра `C` для `LogisticRegression` через кросс-валидацию и `LogisticRegressionCV` по метрике ROC-AUC.
    *   **Градиентный бустинг:** применение `CatBoost` (без ручного кодирования категорий), тестирование стандартных параметров и подбор гиперпараметров для улучшения результата.
    *   Отправка лучшего предсказания в Stepik.

<summary><strong>HW 4: Полносвязные и свёрточные нейронные сети на PyTorch</strong></summary>

*   **Тема:** Практика построения нейронных сетей в `PyTorch`: от реализации логистической регрессии вручную до свёрточных сетей.
*   **Задания:**
    *   **Часть 1. Датасет Moons:**
        *   Реализация кастомного модуля логистической регрессии ($logits = XW + b$) без использования `nn.Linear`; инициализация весов нормальным распределением.
        *   Реализация полного цикла обучения с оптимизатором и функцией потерь BCE; оценка accuracy на тестовой выборке (~86.9%).
    *   **Часть 2.1. MNIST — полносвязные сети:**
        *   Реализация двухслойной полносвязной сети (`nn.Sequential`, 128 нейронов, активация ELU) через стандартный train/val/test loop.
        *   Сравнение функций активации (ReLU, Leaky ReLU, ELU) на графиках validation accuracy/epoch; лучший результат — ELU.
    *   **Часть 2.2. MNIST — свёрточные сети:**
        *   Исследование ядер свёртки: размытие, резкость, выделение горизонтальных/вертикальных границ и тождественное преобразование.
        *   Реализация архитектуры **LeNet** (2 свёртки 3×3, MaxPooling 2×2, 3 полносвязных слоя) в модульном стиле без `Sequential`; accuracy после обучения ~99%.

</details>

    *   Замена последнего классификационного слоя под нужное количество классов (персонажей); дообучение (fine-tuning) на целевом датасете.
    *   Обучение, оценка качества на валидации, визуализация матрицы ошибок.

</details>

<summary><strong>HW 6: Семантическая сегментация медицинских снимков</strong></summary>

*   **Тема:** Решение задачи бинарной сегментации дерматоскопических изображений с использованием архитектур SegNet и U-Net.
*   **Задания:**
    *   **Подготовка данных:** загрузка 200 пар изображений (снимок + маска поражения), ресайз до 256×256, нормализация, разбиение на train/val/test (100/50/50).
    *   **Метрика IoU:** реализация Intersection over Union для оценки качества сегментации.
    *   **BCE Loss:** аналитический вывод численно стабильной формулы лосса, реализация `bce_true()` и `bce_loss()`, сравнение с реализацией `PyTorch`.
    *   **SegNet:** реализация архитектуры (encoder-decoder с MaxPooling/Unpooling), написание функций обучения и инференса, сохранение лучшего чекпоинта.
    *   **Дополнительные лоссы:**
        *   **Dice Loss** на основе коэффициента Дайса с $\epsilon$-сглаживанием для численной стабильности.
        *   **Focal Loss** для работы с несбалансированными классами (пиксели фона vs. поражения).
    *   **U-Net:** реализация архитектуры со skip-connections (конкатенация feature maps энкодера и декодера) по оригинальной статье Ronneberger et al.
    *   **Сравнительный анализ:** обучение SegNet и U-Net на трёх лоссах (BCE, Dice, Focal), сравнение по скорости сходимости и метрике IoU, визуальное сравнение предсказаний.
    *   **BONUS:** реализация дополнительной функции потерь из научной статьи (Boundary Loss / Tversky Loss / аналог).

</details>
