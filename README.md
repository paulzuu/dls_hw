# Семестр 1: Machine Learning & Computer Vision
## Структура репозитория

Каждый проект размещён в отдельной директории:

- `hw1_game_of_thrones/` - Классификация с помощью KNN и sklearn: «Игра Престолов».
- `hw2_linear_models/` - Линейные модели: реализация логистической регрессии с нуля.
- `hw3_kaggle/` - Соревнование Kaggle: полный ML-пайплайн.
- `hw4_conv_cnn/` - Полносвязные и свёрточные нейронные сети на PyTorch.
- `hw5_simpsons_classification/` - Классификация изображений: «Симпсоны».
- `hw6_semantic_segmentation/` - Семантическая сегментация медицинских снимков.

Ниже представлено краткое описание каждого задания и выполненных в его рамках задач.

<summary><strong>HW 1: Библиотека sklearn и классификация с помощью KNN — «Игра Престолов»</strong></summary>

*   **Тема:** Знакомство с полным ML-пайплайном на задаче бинарной классификации: предсказание выживаемости персонажей из вселенной «Игры Престолов».
*   **Задания:**
    *   **Часть 1. Анализ и предобработка данных:**
        *   Исследование пропущенных значений: визуализация через барчарты и тепловые карты (`seaborn`), удаление неинформативных столбцов, заполнение медианой и модой.
        *   Создание новых признаков: `isPopular` (бинаризация popularity score), `boolDeadRelations` (наличие мёртвых родственников), объединение схожих значений категории `culture`.
        *   Кодирование категориальных признаков с помощью `OneHotEncoder`.
        *   Анализ распределения целевой переменной, матриц корреляций и влияния признаков (violin plots, heatmap, pairplot).
        *   Разбиение на train/test с применением идентичных преобразований к тестовой выборке.
    *   **Часть 2. Обучение моделей:**
        *   Обучение 8 классификаторов из `sklearn`: `LogisticRegression`, `DecisionTreeClassifier`, `AdaBoostClassifier`, `RandomForestClassifier`, `GaussianProcessClassifier`, `GaussianNB`, `KNeighborsClassifier`, `SVC`.
        *   Подбор гиперпараметров для `DecisionTreeClassifier` и `AdaBoostClassifier` через `GridSearchCV`.
    *   **Часть 3. Оценка качества:**
        *   Сравнение всех моделей по метрике `accuracy`. Лучший результат показал `GaussianProcessClassifier` (~0.7789 на публичном тесте).
        *   Формирование файла `submission.csv` для сдачи.

<details>
<summary><strong>HW 2: Линейные алгоритмы — реализация логистической регрессии с нуля</strong></summary>

*   **Тема:** Глубокое понимание градиентного спуска и линейных моделей путём реализации с нуля на `NumPy`.
*   **Задания:**
    *   **Градиентный спуск:** реализация функции спуска для $f(x,y) = \sin^2x + \sin^2y$; аналитический расчёт частных производных; визуализация траектории на интерактивном 3D-графике (`plotly`) и графике loss-per-step.
    *   **Генератор батчей:** реализация генератора для SGD с перемешиванием датасета по эпохам (`np.random.permutation`); последний неполный батч отбрасывается.
    *   **Логистическая регрессия:** реализация класса `MyLogisticRegression` с методами `fit` и `get_grad`; функция потерь — бинарная кросс-энтропия; градиент $\frac{\partial L}{\partial w} = X^T(p - y)$.
    *   **ElasticNet-регуляризация:** расширение через наследование — класс `MyElasticLogisticRegression` с L1- и L2-регуляризацией; переопределяется только метод `get_grad`.
    *   **Тестирование:** проверка на синтетических данных и на датасете MNIST (бинарная классификация цифр 0 и 1), оценка через кросс-валидацию.

</details>

<details>
<summary><strong>HW 3: Решение соревнования на Kaggle — полный ML-пайплайн</strong></summary>

*   **Тема:** Самостоятельное решение задачи бинарной классификации в соревновательном формате с оценкой по метрике ROC-AUC.
*   **Задания:**
    *   **Загрузка и первичный анализ данных:** просмотр случайных строк, поиск и обработка пропущенных значений (`isna()`, `info()`), визуализация доли пропусков по столбцам.
    *   **Анализ данных (EDA):** гистограммы и boxplot для числовых признаков; bar- и pie-диаграммы для категориальных; анализ сбалансированности целевой переменной.
    *   **Линейные модели:** нормализация числовых признаков, OneHotEncoding категориальных; подбор гиперпараметра `C` для `LogisticRegression` через кросс-валидацию и `LogisticRegressionCV` по метрике ROC-AUC.
    *   **Градиентный бустинг:** применение `CatBoost` без ручного кодирования категориальных признаков; тестирование дефолтных параметров и подбор гиперпараметров для улучшения скора.
    *   Отправка лучшего предсказания в систему проверки.

</details>

<details>
<summary><strong>HW 4: Полносвязные и свёрточные нейронные сети на PyTorch</strong></summary>

*   **Тема:** Практика построения нейронных сетей в `PyTorch`: от реализации логистической регрессии вручную до свёрточных сетей типа LeNet.
*   **Задания:**
    *   **Часть 1. Датасет Moons:**
        *   Реализация кастомного модуля логистической регрессии ($logits = XW + b$) без `nn.Linear`; инициализация весов нормальным распределением.
        *   Реализация цикла обучения с оптимизатором и BCE-лоссом; accuracy на тесте ~86.9%.
    *   **Часть 2.1. MNIST — полносвязные сети:**
        *   Двухслойная полносвязная сеть (128 нейронов, `nn.Sequential`) с полноценным train/val/test loop.
        *   Сравнение функций активации (ReLU, Leaky ReLU, ELU) на графиках validation accuracy/epoch; лучший результат показала ELU.
    *   **Часть 2.2. MNIST — свёрточные сети:**
        *   Исследование ядер свёртки: размытие, повышение резкости, выделение горизонтальных и вертикальных границ, тождественное преобразование.
        *   Реализация архитектуры **LeNet** (2 свёртки 3×3, MaxPooling 2×2, 3 полносвязных слоя) в модульном стиле без `Sequential`; accuracy после обучения ~99%.

</details>

<details>
<summary><strong>HW 5: Классификация изображений — «Симпсоны»</strong></summary>

*   **Тема:** Многоклассовая классификация изображений на датасете с персонажами «Симпсонов»; от простой CNN до соревнования на Kaggle.
*   **Задания:**
    *   **Подготовка данных:** реализация кастомного `SimpsonsDataset` с `LabelEncoder` для преобразования строковых меток; приведение изображений к единому размеру 224×224, нормализация по статистикам ImageNet (`ToTensor`, `Normalize`).
    *   **Базовая CNN:** построение свёрточной сети глубиной 5 блоков с прогрессивным увеличением числа фильтров (8→16→32→64→96); цикл обучения и формирование первого сабмита.
    *   **Улучшение модели:**
        *   EDA и предобработка: фильтрация некорректных меток, анализ дисбаланса классов.
        *   Аугментации (`RandomHorizontalFlip`, `RandomRotation`, `ColorJitter`) и балансировка классов.
        *   Подбор гиперпараметров: learning rate, batch size, оптимизатор.
    *   **Оценка и анализ ошибок:** вычисление метрики `F1-score` на валидационной выборке; визуализация уверенности модели в предсказаниях; анализ классов, которые сеть путает между собой.

</details>

<details>
<summary><strong>HW 6: Семантическая сегментация медицинских снимков</strong></summary>

*   **Тема:** Бинарная сегментация дерматоскопических изображений с использованием архитектур SegNet и U-Net; сравнение функций потерь.
*   **Задания:**
    *   **Подготовка данных:** загрузка 200 пар изображений (снимок + маска поражения), ресайз до 256×256 (`skimage.transform.resize`), нормализация, разбиение на train/val/test (100/50/50).
    *   **Метрика IoU:** реализация Intersection over Union для пиксельной оценки качества сегментации.
    *   **BCE Loss:** аналитический вывод численно стабильной формулы лосса; реализация `bce_true()` и `bce_loss()` с проверкой через `assert` относительно реализации PyTorch.
    *   **SegNet:** реализация encoder-decoder архитектуры с MaxPooling/Unpooling; написание функций обучения и инференса; сохранение лучшего чекпоинта.
    *   **Дополнительные функции потерь:**
        *   **Dice Loss** — на основе коэффициента Дайса с $\epsilon$-сглаживанием для численной стабильности ($\epsilon = 10^{-8}$).
        *   **Focal Loss** — для работы с несбалансированными классами (пиксели фона vs. области поражения) по оригинальной статье Lin et al.
    *   **U-Net:** реализация архитектуры со skip-connections — конкатенация feature maps энкодера и декодера по статье Ronneberger et al.
    *   **Сравнительный анализ:** обучение SegNet и U-Net на трёх лоссах (BCE, Dice, Focal); сравнение скорости сходимости, значений IoU и визуальных предсказаний.
    *   **BONUS:** реализация дополнительной функции потерь из научной статьи (Boundary Loss / Tversky Loss).

</details>

## Как использовать

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/[Your-Username]/[Your-Repo-Name].git
    ```
2.  Перейдите в директорию первого семестра:
